# round28

## 本轮目标

- 保持 `decision manager` prompt 与 evidence schema 完全不变，只通过 `decision_rules.py` 继续修复 `round27` 剩余的过度保守映射。
- 本轮唯一实验假设是：
- `中性 + 暂不参与` 里有一小批样本，本质上仍属于“高波动消化期中的等待型偏多”，不应继续机械落到 `hold`。

## 跨标的总结

- 这轮是一次明确有效的 rule-only 改善。统一口径下，全主题 `good / bad / unclear = 32 / 33 / 11`，方向准确率 **49.2%**。
- 这意味着：
- 明显高于 `round27` 的 **47.0%**
- 追平 `round11` 的 **49.2%**
- 仅略低于 `round13` 的 **49.3%**
- 分标的结果为：
- `MU`: `10/11/5`，**47.6%**
- `AAOI`: `12/13/1`，**48.0%**
- `2513.HK`: `10/9/5`，**52.6%**
- 这轮最重要的结论不是单一 ticker 被拉高，而是三只票一起站上了更高平台；其中 `2513.HK` 首次在 rule-based 路线上稳定超过 `50%`。

## 结构化归因观察

- 三只票继续全部 `decision_source = rule_based`，说明这轮提升确实来自规则层，不是回退到 prompt 拍板。
- `公司催化趋势型` 的准确率提升到 **55.6%**，继续保持为最强分型桶。
- `Beta情绪技术驱动型` 的准确率进一步升到 **70.0%**，说明上一轮开始的“中性高 beta 样本不应机械压成 hold”方向仍然有效。
- `方向偏向 = 中性` 的准确率继续抬升到 **41.2%**，已经明显高于 `round26` 的 **16.7%**。
- 当前最大拖累已经重新收敛到两组：
- `震荡等待确认`：**20.0%**
- `等待确认`：**16.7%**
- 这说明当前 rule-based 路线的剩余主要问题，不再是 `wait_pullback` 或大面积 `no_entry`，而是少量 `中性 + 等待确认` 仍被过早压成 `hold`。

## 当前结论

- `round28` 可以视为当前“证据层 + 规则决策层”路线的新架构 baseline。
- 它已经基本追平旧路线最强结果：
- `round11`: **49.2%**
- `round13`: **49.3%**
- `round28`: **49.2%**
- 下一步若继续推进，更合理的策略仍然是保持 prompt 与 evidence schema 不动，只做更窄的 rules-only 试验，优先验证 `中性 + 等待确认 + 催化后高波动整理型` 这类剩余坏样本是否应从 `hold` 拉回 `buy_on_pullback`。
