# round26

## 本轮目标

- 在保持主题目标 `55%` 不变的前提下，验证新的“LLM 证据层 + 代码规则决策层”是否能替代纯 prompt 动作拍板。
- 具体来说，本轮希望确认：
- `decision manager` 新增的 6 个证据字段能否稳定进入报告与 parser；
- `decision_rules.py` 是否能基于证据卡独立输出 `buy / hold / sell`；
- 在不依赖 `report_action_fallback` 的情况下，方向准确率是否有机会超过 `round24`。

## 跨标的总结

- 本轮没有成为新 baseline。按统一口径，`round26` 的 `good / bad / unclear = 27 / 39 / 10`，方向准确率 **40.9%**，低于 `round24` 的 **47.0%**，也明显低于 `round13` 的 **49.3%**。
- 分标的结果为：
- `MU`: `10/11/5`，**47.6%**
- `AAOI`: `9/15/2`，**37.5%**
- `2513.HK`: `8/13/3`，**38.1%**
- 这轮最重要的正面结论不是主指标，而是架构验证本身跑通了：三只票的 `signals.csv` 全部是 `decision_source = rule_based`，说明这次结果确实来自“证据卡 + 规则层”，不是又退回到报告动作回退。
- 但第一版规则也暴露出非常清楚的新问题：
- `方向偏向 = 中性` 的准确率只有 **16.7%**，对应 `震荡等待确认 -> 保持观望` 的准确率只有 **19.0%**，说明当前规则把太多高波动样本过早压回了 `hold`。
- `催化后高波动整理型` 的准确率只有 **24.1%**，显著拖累了 `AAOI` 和 `2513.HK`。
- `参与姿态` 只产出了 `等待回调 / 等待确认 / 暂不参与`，没有出现 `可直接参与`，说明这版 evidence schema 还没把“高置信度可积极参与”与“偏多但更像等待型参与”真正拉开。
- 当前判断是：这条“证据层 + 规则层”路线值得继续，但第一版规则仍然过于保守，尤其是对 `催化后高波动整理型` 和 `Beta情绪技术驱动型`。
