## 本轮目标

- 验证新的 evidence schema 与规则层，是否能在 `MU` 这类公司催化趋势样本上保持接近 `round24` 的方向判断质量。

## 改动内容

- `decision manager` 新增 6 个证据字段：`方向偏向 / 趋势完整性 / 催化状态 / 位置状态 / 风险状态 / 参与姿态`。
- parser 现在优先根据证据卡走 `decision_rules.py` 推导动作，而不是直接采用报告里的 `建议行动`。

## 观察结论

- `MU` 本轮为 `10/11/5`，准确率 **47.6%**，和 `round24` 的 **47.8%** 基本持平，说明新架构在这类“公司催化趋势型”样本上并没有立刻失真。
- 但动作仍明显偏向 `buy_on_pullback`：`17` 个买入、`4` 个 hold、`5` 个 sell，平均收益 `-5.4881`，说明证据层虽然能维持方向判断，但还没有把“该更积极买”与“该先防守”充分拉开。
- `MU` 的证据卡分布里，`方向偏向=看多` 有 `17` 次，`参与姿态=等待回调` 也有 `17` 次，而 `可直接参与` 为 `0`。这说明当前 schema 几乎把所有偏多样本都压到了“等待型参与”，还没有产生真正高置信度的积极买入分支。

## 下一步

- `MU` 不是 round26 的主拖累项，后续不需要优先围着它做大改。
- 更合理的方向是保留 `MU` 当前规则骨架，优先去修 `AAOI / 2513.HK` 被过度压成 `中性 / 暂不参与` 的问题。
