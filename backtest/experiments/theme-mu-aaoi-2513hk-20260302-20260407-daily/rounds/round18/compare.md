- 从 `round13` 重新出发，验证一个更窄的 MU 类假设：

## 跨标的总结

- round18 最终是 `good/bad/unclear = 24/40/12`，方向准确率 **37.5%**，明显低于 `round13` 的 **49.3%**。
- `MU` 没有被修好，反而从 `8/12/6` 变成 `8/11/7`。少量好转存在，例如：
- `2026-03-03`：`hold bad_case -> sell good_case`
- `2026-03-10`：`hold bad_case -> buy_on_pullback good_case`
- 但新的坏样本也被放大了，例如：
- `2026-03-02`：`hold unclear -> buy_on_pullback bad_case`
- `2026-03-09`：`buy_on_pullback good_case -> hold bad_case`
- `AAOI` 被明显拖差，从 `10/14/2` 掉到 `6/17/3`，关键坏模式是多个 `buy_on_pullback good_case` 被重新压成了 `hold bad_case`，如 `2026-03-02`、`2026-03-04`、`2026-03-09`、`2026-03-20`。
- `2513.HK` 也从 `15/8/1` 掉到 `10/12/2`，出现多处“反弹参与 -> hold bad_case”的回撤，如 `2026-03-06`、`2026-03-16`、`2026-03-23`、`2026-03-30`。
- 结论：这条 MU guardrail 仍然过宽，会把 `AAOI / 2513.HK` 一起拖进更多 `hold`，不适合作为下一轮继承版本。
