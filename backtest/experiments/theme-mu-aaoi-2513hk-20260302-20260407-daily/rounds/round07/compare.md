# round07

## 本轮目标

- 重写 `decision manager` 的 prompt，把判断顺序改成“先做纯信号方向判断，再补回测兼容字段”。
- 明确把 `buy / hold / sell` 与价格区间、执行计划拆开，避免因为“计划写得不够细”而机械压成 `hold`，也避免因为“能写出参与区”就机械升成 `buy`。
- 观察信号优先 prompt 是否能在不继续放大执行噪声的前提下，改善跨标的纯信号方向质量。

## 跨标的总结

- round07 是第一轮真正基于“信号优先” prompt 的实验。和 round04 相比，全主题纯信号标签从 `good/bad = 16/49` 提升到 `18/49`，`good/bad` 比例从 `0.327` 升到 `0.367`；这说明把信号层和执行层解耦之后，方向判断本身开始出现可见改善。
- 但这轮改善不是均匀发生的。`2513.HK` 从 round04 的 `4/16` 提升到 `8/13`，是本轮最明确的正向样本；代价是动作分布被明显推向偏空，`sell` 从 `6` 激增到 `16`，实际成交也重新回到 `0`。
- `AAOI` 的纯信号标签回到了 round04 水平，`good/bad = 5/20`；说明 signal-first prompt 至少没有继续伤害这只美股样本，但执行收益仍然恶化到 `-10.0406`，进一步证明“信号方向”和“执行质量”现在已经被更清楚地拆开了。
- `MU` 是本轮的主要副作用样本。`buy` 从 `8` 压到 `4`，`sell` 从 `5` 升到 `7`，纯信号标签从 round04 的 `7/13` 退到 `5/16`。也就是说，这版 prompt 在修复港股样本时，把 `MU` 过度推向了风险主导。
- 结论：如果本轮目标是“只优化纯信号方向”，round07 比 round05/06 更接近正确方向，因为它首次在全主题层面优于 round04；但它还不能成为新 baseline，因为 `MU` 的信号质量退步太明显，而且 `2513.HK` 的改善建立在强烈卖出偏置上。
- 下一轮不应回到旧的价格区间式 prompt，而应保留 signal-first 骨架，只收紧一条新的卖出边界：只有当风险已经从“缺少确认”升级为“结构破坏 / 失效条件触发 / 反弹失败后的明显转弱”时，才允许输出 `建议卖出`。目标是保住 `2513.HK` 的方向改善，同时把 `MU` 从过度偏空中拉回来。
