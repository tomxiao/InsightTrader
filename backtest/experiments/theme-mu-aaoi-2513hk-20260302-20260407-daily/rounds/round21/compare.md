# round21

## 本轮目标

- 不再继续做普通 prompt 微调，而是验证“先分型、后决策”的结构化 decision manager 能否稳定输出可归因字段。
- 在不打断现有 backtest 链路的前提下，把 `样本分型 / 主导驱动 / 趋势完整性 / 风险状态 / 趋势判断` 贯穿到 parser、signals.csv 和汇总里。
- 第一优先级不是刷新准确率，而是回答：分型到底是“分错了”，还是“分对了但动作仍然错了”。

## 跨标的总结

- 这轮完成了架构验证，但没有成为新 baseline。统一口径下，全主题 `good/bad/unclear = 24/44/8`，方向准确率 `35.3%`，显著低于 `round13` 的 `49.3%`。
- `MU` 为 `8/14/4`，准确率 `36.4%`；`AAOI` 为 `7/17/2`，准确率 `29.2%`；`2513.HK` 为 `9/13/2`，准确率 `40.9%`，并重新掉回 `0` 成交。
- 这轮最有价值的产出不是主指标，而是新增了按 `样本分型` 与 `趋势判断` 的归因视角，见 [compare_summary.md](/D:/CodeBase/InsightTrader/backtest/experiments/theme-mu-aaoi-2513hk-20260302-20260407-daily/rounds/round21/compare_summary.md) 与 [scenario_summary.json](/D:/CodeBase/InsightTrader/backtest/experiments/theme-mu-aaoi-2513hk-20260302-20260407-daily/rounds/round21/scenario_summary.json)。

## 结构化层观察

- `趋势判断=震荡等待确认` 一共出现 `40` 次，但只有 `6` 个好样本、`30` 个坏样本，准确率仅 `16.7%`。这说明当前结构化层最大的新增偏差不是“分型词不准”，而是把太多样本推回了 `hold`。
- `样本分型=公司催化趋势型` 占了 `40` 个信号，准确率只有 `38.2%`；其中 `AAOI` 有 `21/26` 天都被分进这一类，说明“催化后高波动整理型”并没有稳定接住 AAOI 的主语境。
- `2513.HK` 虽然有 `9` 天被分到 `Beta情绪技术驱动型`，但还有 `12` 天被分到 `催化后高波动整理型`，结果动作分布重新变成 `buy=4 / hold=15 / sell=5`，说明分型没有把港股 beta 语境稳定锁住。
- `MU` 在 `公司催化趋势型` 与 `催化后高波动整理型` 之间来回切换（`16/10`），而不少坏样本恰好出现在“公司催化趋势型 -> 震荡等待确认 -> 保持观望”的链路上，说明分型之后的状态映射仍会把健康回调压成 `hold`。

## 当前结论

- 结构化字段链路已经跑通，这一步是成功的：新字段能稳定进入最终报告、被 parser 读取、落入 `signals.csv`，并在实验汇总中形成新的归因表。
- 但“显式分型 + 当前这版状态映射”作为全局第一层判断，明显会扩大 `hold` 滥用，因此它不能直接替代 `round13` 的主决策骨架。
- 这轮更像一个“架构可行性 round”，证明我们后续可以沿结构化归因继续做实验，但下一轮不应该继续照搬 round21 的全局分型写法。
