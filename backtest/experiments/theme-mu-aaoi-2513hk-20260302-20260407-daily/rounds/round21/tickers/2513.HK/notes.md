## 本轮目标

- 验证 `2513.HK` 是否能被结构化识别为 `Beta情绪技术驱动型`，从而避免再被公司催化逻辑或美股样本的 guardrail 拖偏。

## 改动内容

- 新增显式结构化字段，并把这些字段一路落到 `signals.csv` 与主题汇总，便于判断港股样本到底是“分型错”还是“动作错”。
- 维持 `round13` 的 analyst 输入不变，只观察 decision manager 的新结构化层带来的影响。

## 观察结论

- `2513.HK` 本轮为 `good/bad/unclear = 9/13/2`，准确率 `40.9%`，远低于 `round13` 的 `65.2%`，并重新掉回 `0` 成交。
- 分型只部分成功：`Beta情绪技术驱动型=9`、`催化后高波动整理型=12`、`公司催化趋势型=3`。也就是说，beta 语境并没有被稳定锁住。
- 动作分布重新变成 `buy=4 / hold=15 / sell=5`，其中 `hold` 明显偏多，说明结构化后又把不少原本可交易的样本压回了观望。
- 典型坏样本集中在两类：
- `催化后高波动整理型 -> 震荡等待确认 -> hold`，如 `2026-03-03`、`2026-03-06`、`2026-03-09`、`2026-03-16`
- `Beta情绪技术驱动型 -> 风险主导 -> sell` 或 `hold`，如 `2026-03-13`、`2026-03-19`
- 这说明当前结构化层虽然把一部分港股样本识别成了 beta 型，但后续动作映射仍然过于保守，没有把“高 beta 但仍可顺势参与”的语境保住。

## 下一步

- 后续若继续沿结构化路线推进，`2513.HK` 更适合把 `Beta情绪技术驱动型` 当作局部放宽条件，而不是先分型再统一走一套 `trend/hold/sell` 映射。
- 港股样本最需要保住的是“技术结构与 beta 驱动本身就足以支持偏多”，而不是继续把它送回需要公司催化确认的框架里。
