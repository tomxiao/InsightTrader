# round27

## 本轮目标

- 在不修改 prompt、不修改 evidence schema 的前提下，只通过调整 `decision_rules.py` 修复 `round26` 的过度保守映射。
- 重点验证两条假设：
- `看多 + 等待回调 + 高风险` 不应被机械压成 `hold`；
- `中性 + 等待回调` 在部分高波动样本中，可能更接近 `buy_on_pullback`，而不是默认 `hold`。

## 跨标的总结

- 本轮是一次明确有效的 rule-only 改善。按统一口径，`round27` 的 `good / bad / unclear = 31 / 35 / 10`，方向准确率 **47.0%**。
- 这意味着：
- 明显高于 `round26` 的 **40.9%**
- 基本追平 `round24` 的 **47.0%**
- 但仍低于 `round13` 的 **49.3%**
- 分标的结果为：
- `MU`: `10/11/5`，**47.6%**，基本不变
- `AAOI`: `12/13/1`，**48.0%**，显著高于 `round26` 的 **37.5%**
- `2513.HK`: `9/11/4`，**45.0%**，显著高于 `round26` 的 **38.1%**
- 最重要的是：三只票仍然全部 `decision_source = rule_based`，说明这轮改善确实来自规则层，而不是回退到 prompt 拍板。
- 归因上，最关键的改善来自：
- `方向偏向 = 中性` 的准确率从 **16.7%** 提升到 **33.3%**
- `趋势延续` 的准确率从 **54.5%** 提升到 **58.9%**
- `Beta情绪技术驱动型` 的准确率从 **45.5%** 提升到 **63.6%**
- `催化后高波动整理型` 的准确率也从 **24.1%** 回升到 **32.1%**
- 当前判断是：`round27` 可以作为“证据层 + 规则决策层”路线的新架构 baseline，但还不能替代 `round13` 作为全主题最佳 candidate。
