# round17

## 本轮目标

- 以 `round13` candidate 为出发点，首次验证“先分型，再决策”的内部判断层能否突破当前接近 `50%` 的平台期。
- 不再继续叠加全局 `buy/hold` guardrail，而是要求 `decision manager` 先区分：
- `公司催化型趋势延续候选`
- `催化后高波动整理型`
- `政策 / 情绪 / 技术 beta 型`
- 然后在对应语境下再决定 `趋势延续 / 震荡等待确认 / 风险主导`。

## 跨标的总结

- round17 最终达到 `good/bad/unclear = 30/36/10`，方向准确率 **45.5%**。
- 这轮没有超过 `round13` 的 **49.3%**，也低于 `round11` 的 **49.2%**；主指标层面属于失败 round。
- `MU` 是明确改善样本，从 `8/12/6` 修回到 `11/10/5`。最关键的修复包括：
- `2026-03-03`：`hold bad_case -> sell good_case`
- `2026-03-13`：`hold bad_case -> buy_on_pullback good_case`
- `AAOI` 是主要代价样本，从 `10/14/2` 掉到 `7/16/3`。最明显的坏模式是 `sell` 分支被完全抹掉，动作分布变成 `buy=14 / hold=12 / sell=0`，把几个 round13 的关键修复又退回了 `hold bad_case`：
- `2026-03-02`：`buy_on_pullback good_case -> hold bad_case`
- `2026-03-09`：`buy_on_pullback good_case -> hold bad_case`
- `2026-03-20`：`buy_on_pullback good_case -> hold bad_case`
- `2513.HK` 也从 `15/8/1` 回落到 `12/10/2`。虽然执行层仍有 `13.1859` 的正平均收益，但信号层重新出现“该参与的反弹被压回 `hold`”的问题，例如：
- `2026-03-23`：`buy_on_pullback good_case -> hold bad_case`
- `2026-03-30`：`buy_on_pullback good_case -> hold bad_case`
- 本轮中途还暴露了一个格式问题：模型把“内部样本分型判断”写进了正文前言，导致 `MU 2026-03-06` 与 `AAOI 2026-04-01` 的 parser 首次失败。我随后补了一条更强的格式约束，并只重生了这两个 summary；这次修正没有改变本轮核心逻辑，只是把输出重新压回 parser 可读结构。
- 结论：**“先分型，再决策”本身有信息量，证明它能明显修回 `MU`，但当前这版分型层仍然过度保护了 `AAOI` 的“催化后整理”语境，并把 `2513.HK` 的一部分 beta 反弹重新压成了 `hold`。**
- 下一轮如果继续沿这条路走，更合理的方向不是放弃分型，而是把分型降级成**非对称 tie-breaker**：
- 保留 `公司催化型趋势延续候选` 对 `MU` 的修复能力；
- 不再把 `AAOI` 一股脑塞进“催化后高波动整理型 -> 更偏 hold”的框架；
- 若要继续用分型，优先只把它做成 `政策 / 情绪 / 技术 beta 型` 的窄例外，而不是全局性的第一判断层。
