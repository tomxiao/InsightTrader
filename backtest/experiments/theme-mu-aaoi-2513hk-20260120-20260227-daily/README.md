# 主题：MU + AAOI + 2513.HK（2026-01-20 ~ 2026-02-27，日频）

## 主题定义

- 样本池：`MU`、`AAOI`、`2513.HK`
- 回测区间：`2026-01-20` ~ `2026-02-27`
- 采样方式：`daily`
- 主题类型：时间外验证
- 调优主线：验证现有 `evidence + rules` 信号决策体系在新时间窗里的稳定性，不在一开始就继续做大幅调优

## 为什么开这个主题

- 上一个主题 [theme-mu-aaoi-2513hk-20260302-20260407-daily](/D:/CodeBase/InsightTrader/backtest/experiments/theme-mu-aaoi-2513hk-20260302-20260407-daily/README.md) 已经在 `round30` 达到 **57.6%**
- 但那个结果仍然只覆盖：
- 同一组标的
- 同一个时间窗
- 在进入价格区间和执行收益优化之前，需要先回答一个更基础的问题：
- `round30` 形成的证据字段与规则决策层，换到新的时间区间后还能否保持较高方向准确率

## 主题目标

- 当前主题的正式主目标不是继续冲更高数字，而是验证跨时间窗稳定性
- 主指标仍统一为：
- `accuracy = good / (good + bad)`
- `unclear` 不计入准确率分母
- 当前验证门槛：
- 若新主题 baseline 能稳定保持在 `50%+`，说明信号体系具备较好的时间外泛化能力
- 若仍能维持在 `55%` 附近，则说明上一个主题的突破大概率不是单窗口偶然性

## 起始基线

- 本主题已经建立了自己的 baseline：`round01`
- `round01` 全主题纯信号统计：`good = 35`，`bad = 28`
- 对应方向准确率：`35 / (35 + 28) = 55.6%`
- 启动时的外部参考基线为：
- 来源主题：`theme-mu-aaoi-2513hk-20260302-20260407-daily`
- 参考 round：`round30`
- 参考准确率：`57.6%`
- 当前结论是：新时间窗下的 baseline 已经非常接近来源主题的 `57.6%`

## round01 结果

- 类型：`full_round`
- 作用：建立新时间窗下的完整 baseline
- 结果：
- 全主题 `35 / 28 / 19`，方向准确率 **55.6%**
- `MU`: `10 / 7 / 11`，准确率 **58.8%**
- `AAOI`: `16 / 9 / 3`，准确率 **64.0%**
- `2513.HK`: `9 / 12 / 5`，准确率 **42.9%**
- 这说明：
- `round30` 的 `evidence + rules` 决策体系在新时间窗里依然维持了 `55%+`
- 时间外泛化已经初步成立
- 当前新主题的主要短板不在 `MU / AAOI`，而在 `2513.HK`

## round01 计划

- 类型：`full_round`
- 作用：建立新时间窗下的完整 baseline
- 运行要求：
- 必须从头生成 analyst 报告与 decision 报告
- 不复用旧主题的 `1_analysts`
- 默认沿用当前代码里的 `decision manager` prompt、evidence schema 和 `decision_rules.py`

## 成功标准

- `round01` 跑完后，至少要回答这几件事：
- `MU / AAOI / 2513.HK` 在新时间窗里是否仍保持相近的结构分布
- `中性 / 等待确认 / 暂不参与` 这些在旧主题里被系统修回的分支，是否在新时间窗依旧稳定
- `round30` 的规则层提升，是否主要依赖旧窗口里的局部模式

## 后续分支

- 如果本主题 baseline 维持在 `50%+`
- 下一步优先考虑开“换标的不换区间”的第二个验证主题
- 如果本主题 baseline 明显低于 `50%`
- 下一步优先回到规则层做泛化修正，而不是进入价格区间与执行质量阶段
- 当前实际结果已经满足 `50%+`，并且直接达到 `55.6%`
- 因此更合理的下一步有两条：
- 继续开“换标的不换区间”的第二个验证主题
