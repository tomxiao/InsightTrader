# 对话式移动端前端一期低保真原型 v1

## 1. 说明

本文档用于描述移动端一期“统一对话页 + 左侧抽屉 + 底部账户菜单”的文本低保真原型，便于产品、设计、前端和后端统一页面结构认知。

本文仅描述：

- 页面结构
- 信息层级
- 模块职责
- 关键交互

本文不涉及：

- 视觉风格细节
- 品牌色和字体规范
- 最终 UI 稿尺寸标注

## 2. 原型范围

本次低保真原型覆盖以下内容：

1. 登录页
2. 当前会话页
3. 左侧抽屉
4. 底部账户菜单
5. 完整报告查看页

## 3. 页面关系

整体关系如下：

1. 用户先进入登录页
2. 登录成功后进入当前会话页
3. 在当前会话页中通过左上角按钮打开左侧抽屉
4. 通过抽屉新建会话或切换历史会话
5. 通过完整报告卡片进入完整报告查看页

## 4. 登录页

### 4.1 页面目标

让用户快速完成登录并进入当前会话页。

### 4.2 页面结构

```text
+--------------------------------------------------+
|                    Logo / 产品名                 |
|                                                  |
|              一句话产品说明或欢迎语              |
|                                                  |
|  [ 用户名 / 邮箱 / 手机号 ]                      |
|  [ 密码 ]                                        |
|                                                  |
|  [ 登录按钮 ]                                    |
|                                                  |
|  登录提示 / 错误提示                             |
+--------------------------------------------------+
```

### 4.3 交互说明

- 登录成功后直接进入当前会话页
- 不增加额外首页跳转

## 5. 当前会话页

### 5.1 页面目标

作为默认落点和唯一主工作区，承载分析发起、处理中反馈、结果展示和继续追问。

### 5.1.1 一期规则

- 每个用户同一时间只允许存在 1 个进行中的分析任务
- 任务运行中用户可继续编辑输入框，但新输入仅保留为草稿
- 对话线程中的报告相关内容只保留两种卡片：执行摘要卡片、完整报告卡片

### 5.2 空白会话态

```text

| [菜单]              新会话                       |
+--------------------------------------------------+
|                                                  |
|            你好，今天想分析哪只股票？            |

| [ 输入框：输入股票名称、代码或问题... ] [发送]   |
+--------------------------------------------------+
```

### 5.5 当前会话页模块拆解

#### 顶部栏

- 左上为抽屉按钮
- 中间为当前会话标题
- 右上可暂时留空
- 任务运行中在顶部下方显示轻量状态条

#### 消息流区域

- 用户消息
- 系统消息
- 状态消息
- 执行摘要卡片
- 完整报告卡片
- 错误提示

#### 底部输入区

- 多行文本输入框
- 发送按钮
- 可选快捷问题
- 任务运行中发送动作仅保留草稿，不触发新的正式请求

## 6. 左侧抽屉

### 6.1 页面目标

承载新建会话、历史会话切换和账户入口，不与主会话页面并列竞争。

### 6.2 抽屉结构

```text
+--------------------------------------+
| Logo / 产品名                        |
|                                      |
| [ + 开启新对话 ]                     |
|                                      |
| 今天                                 |
| - 宁德时代长期持有分析         [···] |
| - 腾讯控股风险复盘             [···] |
|                                      |
| 昨天                                 |
| - 比亚迪财报解读               [···] |
|                                      |
| 近 7 天                              |
| - 茅台估值讨论                 [···] |
| - 港股科技板块观点             [···] |
|                                      |
|                                      |
| [头像] tom肖                    [···]|
+--------------------------------------+
```

### 6.3 交互说明

- 点击 `+ 开启新对话`：创建新会话并切换到空白线程
- 点击历史会话：进入对应线程
- 点击底部用户区域或更多按钮：弹出账户菜单
- 若已有进行中的分析任务，其他会话中不能再发起新的分析任务

### 6.4 抽屉设计原则

- 抽屉主区直接显示历史会话列表，而不是只放一个“历史对话”按钮
- 历史列表尽量按时间分组
- 一级结构保持轻量，不做多层嵌套导航

## 7. 底部账户菜单

### 7.1 目标

以轻量菜单替代独立“我的页”。

### 7.2 展开形态

```text
+--------------------+
| 系统设置           |
| 帮助与反馈         |
| 退出登录           |
+--------------------+
```

### 7.3 说明

- 一期不强调复杂资料编辑
- 一期不单独做重型“我的页”
- 若后续设置项增加，可从 `系统设置` 进入独立次级页面

## 8. 完整报告查看页

### 8.1 页面目标

承载完整分析内容阅读。

### 8.2 页面结构

```text
+--------------------------------------------------+
| [返回]            完整报告                       |
+--------------------------------------------------+
| 股票：宁德时代 / 300750                          |
| 执行摘要                                         |
| ...                                              |
|                                                  |
| 报告正文                                         |
| ...                                              |
+--------------------------------------------------+
```

### 8.3 交互说明

- 从完整报告卡片进入
- 查看后可返回原会话继续追问
- 一期不在会话内直接展开完整报告正文

## 9. 关键状态说明

### 9.1 空状态

- 新会话刚创建，等待输入
- 可展示一句欢迎语和少量快捷问题

### 9.2 处理中状态

- 使用消息流中的状态消息提示，不另开页面
- 状态按时间顺序插入线程中
- 顶部状态条持续显示当前步骤、已用时间和预计剩余时间
- 输入框可继续编辑，但发送后仅保留草稿
- 提供取消任务或返回当前任务的动作反馈

### 9.3 完成状态

- 展示执行摘要卡片和完整报告卡片
- 保留输入框，方便继续追问
- 执行摘要支持折叠 / 展开原文

### 9.4 失败状态

- 在消息流中给出失败说明
- 提供重试建议或重新发起入口

## 10. 设计与实现备注

### 10.1 产品侧

- 重点确保“当前会话页”是绝对主场景
- 不再强化“首页”和“我的页”的独立感知

### 10.2 设计侧

- 抽屉与主会话之间的主次关系要明确
- 底部输入区必须优先保证可用性
- 执行摘要卡片和完整报告卡片需要明显区别于普通消息气泡
- 完整报告正文不进入聊天流

### 10.3 前端侧

- 抽屉切换会话时要保持线程恢复体验
- 需要处理键盘弹起对输入区和消息流的影响
- 需要考虑长消息、长报告和网络波动场景

## 11. 结论

本低保真原型对应的产品形态为：

**登录后直接进入当前会话页，左侧抽屉承载新建会话与历史切换，抽屉底部以轻量账户菜单替代独立“我的页”。**

这一形态适合移动端一期快速收敛，也更贴合“对话式个股分析 + 报告解读”的核心目标。
