# 移动端追问回复优化 TODO（2026-04-16）

## 1. 背景

今天已经完成一版 `insight_reply` 的后端改造，包括：

- `summary_card` 摘要进入 `ReportInsightContext`
- `ReportInsightAgent` 优先参考轻量摘要
- 工具调用提示词改成“优先直接回答、按需读章节”
- 最终回答阶段禁止继续调用工具
- 增加轻量后处理，去掉“回答：/直接回答：”等前缀

但联调后的实际体验仍然没有达到移动端对话目标。

## 2. 当前问题

### 2.1 回答时间过长

- 用户追问经常超过 10 秒才返回
- 对移动端聊天场景来说，等待明显偏长
- 现在的 Agent 虽然做了“探索阶段 + 最终回答阶段”分离，但整条链路仍然偏重

### 2.2 回答内容过长

- `insight_reply` 仍然常常是一屏看不完
- 实际输出仍然保留明显的“报告体”
- 常出现：
  - 开头铺垫过长
  - 多级小标题
  - 过多要点枚举
  - 主动展开过多解释

### 2.3 当前方案的根本问题

目前的问题不是“LLM Agent 路线本身走不通”，而是当前实现仍然把追问回复当成“报告解读”，而不是“聊天推进”。

现在的实现更像：

- 尽量完整回答
- 尽量多给依据
- 尽量一次说透

而移动端真正需要的是：

- 先短答
- 只回答当前最核心一层
- 留出自然追问入口
- 用户继续追问，再展开下一层

## 3. 已达成的结论

### 3.1 不采用“深路径”思路

不采用“简单问题走快路径，复杂问题走深路径”的产品体验模式。

原因：

- 从对话角度看，系统不应该主动进入“深度完整展开”
- 用户应该通过多轮追问逐步展开，而不是一次读完半篇报告
- 即使问题复杂，也应该优先输出“当前轮最小充分回答”

### 3.2 目标改为“短答 + 引导追问”的循环

`insight_reply` 的理想形态应为：

- 第一行直接回答
- 最多 2-3 个短要点
- 不主动穷举所有维度
- 末尾自然提示可继续追问的方向

示例形态：

```text
主要是三类风险：增长放缓、技术路线变化、资本开支压力。

- 增长放缓：高预期下，一旦增速回落，估值压力会放大。
- 技术替代：新路线可能削弱现有优势。
- 资本压力：扩产和投资同时推进，会压缩资金效率。

如果你愿意，我可以继续展开哪个风险最值得警惕，或者它们会怎么影响股价。
```

### 3.3 继续保留 LLM Agent 路线

不回退为纯规则系统，也不放弃 Agent。

但 Agent 的定位要从：

- “报告解读器”

改成：

- “对话推进器”

即：

- 不追求一次答全
- 不主动写成长报告
- 只为当前轮生成最小充分回答

## 4. 明天的改造方向

### 4.1 调整 Agent 的回答目标

修改 `ReportInsightAgent` 的系统提示词和最终回答阶段要求，明确：

- 默认禁止主动深展开
- 默认禁止报告式分节回答
- 默认只回答当前问题最核心的一层
- 默认保留后续追问空间

### 4.2 增加“短答收口”步骤

在最终答案生成后，增加一层硬收口逻辑，而不是只依赖 prompt：

- 第一行必须直接回答
- 最多保留 3 条要点
- 总长度尽量控制在 120-180 字
- 删除“核心风险 / 关键要点 / 总结”等报告式标题
- 如回答过长，自动二次压缩

### 4.3 增加“引导追问”输出要求

在回答末尾补一条自然的继续追问引导，但不要做成菜单化、产品化按钮文案。

目标：

- 像聊天，不像表单
- 引导用户继续问下一层
- 不一次性把所有材料摊开

### 4.4 进一步控制工具调用

在保持 Agent 路线前提下，进一步减少不必要的章节探索：

- 简单追问优先只基于 `summary_text`
- 仅当摘要不足以回答时才允许读取章节
- 降低“为了完整而主动多读材料”的倾向

### 4.5 评估是否需要二次压缩模型调用

待评估两种实现方式：

1. 仅靠当前回答阶段 prompt + 本地裁剪
2. 增加一次独立的“压缩改写”调用

优先目标：

- 先让体感明显变短
- 再评估是否值得为更稳定的短答增加一次模型调用

## 5. 建议的开发顺序

明天建议按这个顺序推进：

1. 重写 `ReportInsightAgent` 的回答目标与最终阶段提示词
2. 增加统一的 `_shrink_answer(...)` / `_should_shrink_answer(...)`
3. 增加“继续追问引导”模板
4. 联调实际追问样例
5. 根据真实效果再决定是否引入独立压缩调用

## 6. 重点验证样例

明天联调时优先验证这些问题：

- “主要风险是什么？”
- “结论是什么？”
- “现在适合买入吗？”
- “为什么偏谨慎？”
- “最值得关注的一个风险是什么？”

验证标准：

- 回复速度明显改善
- 大多数回答一屏内可读完
- 语言像聊天，不像二次报告
- 用户读完后自然知道下一句该怎么问
