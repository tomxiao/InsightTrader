# F3 浏览器走查评测样本

目标：用于浏览器端逐条输入，观察系统是否能够识别、推断并确认目标股票标的。

## 样本原则
- 覆盖直接 ticker、中文简称、英文公司名、市场提示、歧义消解、上下文补全。
- 每条样本只关注“系统最终识别出的标的是否正确”，不以分析报告完整产出为主要验收目标。
- 若系统进入 `need_confirm` 或 `need_disambiguation`，只要候选集和最终确认标的符合预期，即视为 F3 主目标达成。

## 评测样本

### Case 1
- 输入：`分析苹果，重点看估值`
- 期望标的：`AAPL`
- 期望名称：`Apple Inc.`
- 观察点：中文简称是否能直接映射到美股苹果，并进入确认或直接解析成功

### Case 2
- 输入：`看看腾讯控股最新情况`
- 期望标的：`0700.HK`
- 期望名称：`Tencent Holdings Limited`
- 观察点：中文全称/常用简称是否能映射到港股腾讯

### Case 3
- 输入：`分析 300750.SZ，重点关注未来两季催化`
- 期望标的：`300750.SZ`
- 期望名称：`宁德时代新能源科技股份有限公司`
- 观察点：标准 A 股 ticker 是否能直接解析，并保留关注点

### Case 4
- 输入：`帮我看看英伟达`
- 期望标的：`NVDA`
- 期望名称：`NVIDIA Corporation`
- 观察点：中文海外科技股别名识别

### Case 5
- 输入：`我想看 0700.HK`
- 期望标的：`0700.HK`
- 期望名称：`Tencent Holdings Limited`
- 观察点：直接输入港股 ticker 时是否可跳过歧义

### Case 6
- 输入：`重点看估值和护城河`
- 第二轮输入：`苹果`
- 期望标的：`AAPL`
- 期望名称：`Apple Inc.`
- 观察点：系统是否能在两轮对话中把第一轮关注点与第二轮标的补全合并

### Case 7
- 输入：`分析 920964.BJ`
- 期望标的：`920964.BJ`
- 观察点：北交所 ticker 是否不会触发 catalog 归一化报错，且能被当作有效 A 股标的处理

## 走查记录建议
- 实际输入
- 实际展示状态：`collect_more / need_confirm / need_disambiguation / resolved / failed`
- 实际候选列表
- 最终确认标的
- 是否符合预期
- 异常现象或日志
